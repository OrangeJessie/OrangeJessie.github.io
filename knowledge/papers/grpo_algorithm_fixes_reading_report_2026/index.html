GRPO 算法修正论文阅读报告

阅读范围覆盖 DAPO、Dr. GRPO、VAPO、GSPO、DCPO 与 AReaL。重点是四类问题： 训练不稳定、长度偏置、零梯度、rollout 利用率低。

TL;DR

DAPO：复现 R1-Zero 风格 RLVR 的第一个强开源 recipe，重点修 entropy collapse、零梯度和超长截断噪声。

Dr. GRPO：指出长度归一化与组内 std 归一化会改变优化目标，训练中的 length increase 不一定是能力涌现。

VAPO：重新引入 value model，用 value pretraining、decoupled GAE 和 length-adaptive GAE 提升 long-CoT credit assignment 上限。

GSPO：把 token-level importance ratio 改成 sequence-level ratio，解决 reward 粒度和优化粒度不匹配，尤其适合 MoE。

DCPO：针对固定 clipping 和同奖赏零梯度，提出动态 clipping、平滑 advantage 与 OTM，提高有效 rollout 利用率。

AReaL：从系统侧解决同步 rollout 等待最长样本导致的 GPU 空转，用异步 pipeline 配合 staleness-aware PPO。

02先分清 sequence 和 token level

很多 GRPO 改进看起来都在说 sequence-level、token-level，但它们说的并不总是同一层。最容易混的是三件事： reward/advantage 的粒度、importance ratio/clipping 的粒度、以及最后 loss reduction 的粒度。

层 sequence-level 是什么 token-level 是什么 对应论文问题
Reward / Advantage 整条 response 一个 reward，例如数学题最终答对为 1、答错为 0；GRPO 再把同组 rollout 的 reward 做相对化，得到每条 response 一个 advantage。 每个 token 或每个 reasoning step 有自己的 advantage，例如 value model/GAE 把终点 reward 分配回中间步骤。 VAPO 关心 long-CoT 的 token credit assignment；vanilla GRPO 通常是 sequence advantage 复制到所有 token。
Ratio / Clipping 比较整条 response 在新旧 policy 下的概率变化，再对整条 response 做 clipping。 每个 token 单独算新旧概率比值并 clipping。 GSPO 认为 reward 是整条 response 的，所以 ratio/clipping 也应更偏 sequence-level；DAPO/DCPO 则主要修 token-level clipping 过紧的问题。
Loss Reduction 先得到每条 response 的 loss，再把 response 当训练样本聚合。 把 token loss 展开后按 token 聚合，或只在单条 response 内做 token mean。 DAPO 的 token-level loss、Dr. GRPO 的 length normalization 批评、DCPO 的 OTM，都在改这一层。
最小例子

同一个 prompt 采样三条 response：A 答对、B 答错、C 答对。GRPO 的组内 reward 会让 A/C 拿正 advantage，B 拿负 advantage。

但模型不是一次性“生成整条 response”的黑盒，它是一个 token 一个 token 采样的。因此训练时通常会把 A 的正 advantage 乘到 A 里每个 token 的 logprob loss 上，把 B 的负 advantage 乘到 B 里每个 token 上。

这就是最核心的错位：奖励常常是 sequence-level，动作和梯度却是 token-level。如果不处理好，正确答案里的废话 token 也会被奖励，错误答案里的正确中间步骤也会被惩罚；如果 loss 聚合再做错，长短 response 的权重还会被额外扭曲。

公式里的 A_i,t 怎么理解

GRPO 公式常写成 A_i,t，因为 loss 是逐 token 求和的，每个 token 位置都需要一个 advantage 槽位。

但在只有最终 outcome reward 的 vanilla GRPO 里，通常先得到整条 response 的 reward R_i，再做组内标准化得到一个 scalar A_i。此时实现上会把它广播到该 response 的所有 token：A_i,1 = A_i,2 = ... = A_i。

所以 token loss 仍然不完全一样，因为每个 token 的 probability ratio、clipping 状态和 KL 不同；只是 advantage 本身没有区分这条 response 内哪个 token 更关键。

只有引入 process reward、token/step verifier、value model + GAE 这类机制时，A_i,t 才真正变成随 token 位置变化的 token-level advantage。

参数什么时候更新

token-level loss 不表示“每生成一个 token 就更新一次模型”。GRPO 通常先用旧策略为一批 prompt 各采样 G 条 response，算完 reward 和 advantage，再把这些 response 里的所有 token loss 聚合成一个 batch/minibatch loss。

随后执行一次反向传播，把所有 token loss 对参数的梯度加总起来；优化器在 optimizer.step() 时统一更新参数。也就是说，token 是 loss 项，response/group 是样本组织方式，真正更新发生在 batch/minibatch 级别。

如果实现里有 microbatch 或 gradient accumulation，会先做多次 backward 累积梯度，再做一次 optimizer step；如果 PPO/GRPO 配置了多轮 update epoch，同一批 rollout 数据还可能被复用多次。

一句话记法

GRPO 原始直觉：用同一 prompt 下多条 response 的相对好坏，决定每条 response 里的 token 概率该整体上调还是下调。

后续改进：DAPO/DCPO 修 token clipping 和 token loss 利用率；Dr. GRPO 修长度与 std 归一化带来的目标偏置；GSPO 修 sequence reward 与 token ratio 的粒度错配；VAPO 重新引入 value，让 advantage 更细地回传到 token/step。

DAPO arXiv:2503.14476v2 ByteDance Seed + AIR

DAPO: Decoupled Clip and Dynamic Sampling Policy Optimization

DAPO 的价值在于把“naive GRPO 为什么跑不出 R1-Zero”拆成可复现 recipe。作者报告 Qwen2.5-32B 上 naive GRPO 只有 AIME24 avg@32 30 分， DAPO 最终达到 50 分，并开源了系统、数据与训练代码。

机制原理

Clip-Higher

PPO/GRPO 的 clipping 本意是限制 policy update 的 trust region，但对正 advantage token 来说，上界会不成比例地限制低概率 token。

例如旧策略只给某 token 0.01 概率，普通 1 + eps 上界只能把它推到 0.012；而旧概率 0.9 的常见 token 几乎不受真实限制。

这会让模型更容易强化已经会的 exploitation token，却很难提升偶然探索出的关键 reasoning token。DAPO 只放宽上界、不放宽下界，目标是增加“好但原本低概率”的 token 被学习到的机会，同时避免把低概率 token 直接压到 0 造成采样空间塌缩模型生成时概率分布过早集中到少数高概率 token 或固定推理路径，低概率但可能有价值的探索路径几乎再也采不到。结果是 rollout 多样性下降，模型更难发现新的正确推理方式。。

Dynamic Sampling

GRPO 的 advantage 是同一 prompt 内相对比较。若一组 rollout 全对或全错，所有样本 reward 相同，组内 centered advantage 变成 0，这组 prompt 即使消耗了 rollout，也不给 policy 提供方向。

DAPO 过采样后只保留 0 < #correct < G 的组，本质是在训练后期主动寻找“模型刚好有分歧”的 prompt。这些 prompt 同时包含正负样本，能告诉模型哪些推理路径应该上调、哪些应该下调。

代价是采样成本会上升，并且全对 prompt 的保持能力不再通过这一步训练。

Token-Level Loss

sample-level mean 会让每条 response 在 loss 中权重相同，因此长 CoT 内每个 token 的边际权重更小。

这有两个后果：正确长推理里的关键中间步骤学得慢，错误长输出里的重复、乱码、绕圈也罚得轻。

DAPO 改成 token-level 聚合原 GRPO 常见做法是先在每条 response 内对 token loss 求平均，再对 response 求平均，所以每条 response 权重相同，长答案里的单个 token 权重更小。DAPO 改成把 group 内所有 response 的 token loss 展开后一起求平均，即除以总 token 数；这样每个 token 权重相同，长 response 的总权重按 token 数增加。 后，优化信号跟 token 数量对齐。一个模式只要持续贡献好/坏 reward，就会被按出现的 token 充分强化或抑制，而不是被“这条答案很长”稀释掉。

Overlong Reward Shaping

长 CoT 训练里，截断不等于推理错误。若把所有超长截断都直接判成负样本，模型会把“推理方向有效但表达太长”的 trajectory 当成坏策略，reward noise 会破坏探索。

DAPO 先说明 overlong filtering 能稳定训练，再用 soft overlong punishment 把长度惩罚做成连续区间：没超过期望长度不罚，接近最大长度逐步加罚，真正超过上限才给满惩罚。

它把“正确性”和“长度控制”拆开，避免一个 hard penalty 同时惩罚两件事。

Dr. GRPO arXiv:2503.20783v2 · COLM 2025 Sea AI Lab / NUS / SMU

Understanding R1-Zero-Like Training: A Critical Perspective

这篇的核心不是再堆 recipe，而是指出 GRPO 常见实现存在优化目标偏置。作者把 Dr. GRPO 称为 Group Relative Policy Optimization Done Right。

$$\hat A_i^{\mathrm{GRPO}}=\frac{R(q,o_i)-\mathrm{mean}(R)}{\mathrm{std}(R)},\quad L \propto \frac{1}{|o_i|}\sum_t \ell_{i,t}$$
$$\tilde A_i = R(q,o_i)-\mathrm{mean}(R),\quad \text{Dr. GRPO removes } |o_i| \text{ and } \mathrm{std}(R).$$

偏置原理

长度归一化偏置

GRPO 常见实现会把一条 response 的 token loss 除以该 response 长度。这个操作在监督学习里像是普通 mean，但在 RL 里 response 长度不是常数，而是 policy 自己生成出来的随机变量。

正确答案有正 advantage 时，长答案每个 token 得到的正向更新更小；错误答案有负 advantage 时，长答案每个 token 受到的惩罚也更小。

于是目标函数会隐式偏好“正确答案短一点、错误答案长一点”，训练曲线里的 length increase 可能只是 objective bias，而不一定是模型学会了更深思考。

难度归一化偏置

组内 std(R) 归一化看似只是 advantage normalization，但它是在每个 question 内单独做的。

对 reward 方差很小的 prompt，例如几乎全对或几乎全错的问题，除以很小的 std 会把该 prompt 的权重放大；对 reward 方差较大的 prompt，权重反而较小。

结果是不同难度问题被重新加权，优化目标不再等价于“最大化所有 prompt 的平均可验证 reward”，而混入了由组内方差决定的 difficulty bias。

Dr. GRPO 修正

Dr. GRPO 的修正很克制：移除 response length normalization 和 group std normalization，只保留组内 reward 去均值作为相对 advantage。

论文进一步指出这个形式与 RLOORLOO 是 REINFORCE Leave-One-Out。对同一 prompt 的第 i 条 response，它用“其他 G-1 条 response 的平均 reward”作为 baseline：A_i = R_i - mean(R_j, j≠i)。这样不用 value model，也能用组内其他样本估计相对优势。Dr. GRPO 的 R_i - mean(R_all) 与 RLOO 只差 G/(G-1) 的缩放，可并入 learning rate。 只差一个可并入 learning rate 的缩放因子。它不是新增复杂 trick，而是把实现拉回更接近无偏 policy gradient 的目标。

关键判断：如果训练中看到 response length 上升，不能立即归因于“模型学会深度思考”。必须拆分 correct 与 incorrect 的长度曲线。 Dr. GRPO 的经验结果显示，它能显著压低错误答案长度，同时保持或提升 reasoning benchmark。
VAPO arXiv:2504.05118v3 ByteDance Seed

VAPO: Value-model-based Augmented PPO

VAPO 的立场和 DAPO 相反：不是继续 value-free，而是认为 value model 如果训练好，credit assignment 上限更高。 它在 Qwen2.5-32B 上报告 AIME24 avg@32 约 60.4，超过 DAPO 10 分以上。

机制原理

先分清三种信号

在数学/代码这类 verifiable task 里，训练信号通常只有终点一个 R：答案对就是 1，错就是 0。中间第 10 个、第 100 个 reasoning token 本身没有独立 reward。

PPO 需要的是每个 token 的 advantage：这个 token 相比当前模型平均水平，是应该提高概率还是降低概率。这一步叫 Credit assignment把“最终答对/答错”的全局结果拆回到每个生成动作上。对于 long-CoT，难点是最终 reward 离早期 token 很远，算法必须判断哪些中间推理步骤真正提高了最终正确率。。

reward model/verifier回答“完整答案最后对不对”；value model回答“给定当前 prefix，如果从这里继续生成，最终能拿到多少期望 reward”。后者不是给 prefix 打分，而是在估计从这个 prefix 出发的成功概率。

Value model bias

为什么不能直接拿 reward model 当 value model？因为 reward model 是在完整回答上训练的。一个 prefix 还没写完证明或答案，reward model 往往会给低分；但 value model 要估计的是“这个 prefix 继续写下去之后最终答对的概率”。

举个例子：一条 200 token 的轨迹最后答对了，Monte Carlo 视角下，这条轨迹上每个 prefix 的最终 return 都是 1。早期 prefix 看起来“不完整”，但它并不等于没价值；它可能已经走在正确推理路径上。

如果 reward-initialized value 在早期 prefix 给低分、越接近 <EOS> 越高，GAE: Generalized Advantage EstimationPPO 中常用的 advantage 估计方法。它把从当前位置开始的一串 TD error 按 lambda 衰减加权求和，得到每个 token/state 的 advantage。lambda 越大，看得越远，越接近完整轨迹回报；lambda 越小，越依赖 value model 的一步估计。 里的 TD error: Temporal-Difference Error一步 value 预测误差：delta_t = r_t + gamma V(s_{t+1}) - V(s_t)。在 long-CoT verifier reward 里，中间 r_t 通常为 0，所以它主要比较“生成这个 token 后的新 prefix 价值”与“旧 prefix 价值”的差。 V(s_{t+1})-V(s_t) 会在很多位置变成正数。模型会把“上下文变完整导致 value 上升”误读成“这些 token 都有正贡献”，这就是 VAPO 说的初始化偏置。

VAPO 的 value pretraining 是先固定一个 policy 采样，用最终 reward 作为 Monte Carlo return沿一条完整采样轨迹真正拿到的累计回报。这里中间 reward 为 0、终点 reward 为 0/1，且 gamma=1，所以一条轨迹上任意 prefix 的 sampled return 都等于最终 0/1。多条轨迹平均后就近似“从该 prefix 出发的期望成功率”。 去训练 value。它想让 value 先学会“prefix 到最终正确性”的映射，再让 PPO 用它做 baseline。

GAE 为什么在长 CoT 上失真

GAE 会把每步 TD error 加权求和：越远的未来信号权重越小，约按 lambda^distance 衰减。

$$\delta_t = r_t + V(s_{t+1}) - V(s_t),\quad \hat A_t=\sum_{k\ge 0}\lambda^k\delta_{t+k}$$

long-CoT 的问题是中间几乎没有 reward，真正的 0/1 reward 在终点。若固定 lambda=0.95，距离终点 100 步的终端 reward 权重只有 0.95^100≈0.006，早期 token 几乎看不到最终答对信号。

这时 early token 的 advantage 主要来自 Bootstrapping bias不用等真实最终 reward，而用 value model 对后继状态的估计来更新当前状态。好处是方差低，坏处是如果 value model 本身有偏，偏差会沿长序列一步步传回来。 项，也就是 value model 自己的估计差。如果 value 已经因为 reward-model 初始化而有偏，长序列会把这个偏差放大。

Decoupled GAE

critic/value 的目标是学准：它最好直接贴近真实 return，少被 bootstrap 目标带偏。policy 的目标是更新稳：它不一定要用最高方差的纯 Monte Carlo advantage。

这里的 Decoupled GAE不是指“把共享底座拆成两个模型”或“训完 value 后固定 value”。它指 advantage/target 的计算解耦：critic/value 更新用接近真实 return 的 lambda_critic，policy 更新用另一个 lambda_policy。value pretraining 阶段会用固定 policy 采样来 warm up value；进入 PPO 后，policy 和 value 仍一起训练。 是 GAE 计算目标的解耦，不是冻结 value model。

VAPO 把两者拆开：value 更新侧用 lambda_critic=1.0，更接近 Monte Carlo return，降低 critic 偏差；policy 更新侧单独用 lambda_policy，继续在 bias 和 variance 之间折中。

这解决的是“value 学歪了会污染 advantage，advantage 又反过来带歪 policy”的闭环。

Length-Adaptive GAE

固定 lambda_policy 在长短混合 response 上不合理。短回答如果 lambda 太大，少量随机终端 reward 会让 advantage 方差很高；长回答如果 lambda 不够大，终端 reward 又传不到前面。

VAPO 让 lambda_policy 随输出长度 l 变化，并让 GAE 的有效窗口大致与长度成比例：

$$\sum_{k=0}^{\infty}\lambda_{\mathrm{policy}}^k \approx \frac{1}{1-\lambda_{\mathrm{policy}}}=\alpha l,\quad \lambda_{\mathrm{policy}}=1-\frac{1}{\alpha l}$$

直觉是：长 CoT 需要更长的 credit assignment 窗口，短回答不需要承受那么大的方差。

原理不是“长答案给更多奖励”，而是让长短答案的 credit assignment 尺度一致。

稀疏正样本利用

verifier reward 多数时候只有对/错，正确 rollout 在训练早期很稀少。

VAPO 保留 DAPO 的 Clip-Higher、token-level loss，并加入 positive-example LM loss 和 group sampling，核心是把来之不易的正确轨迹当作可模仿信号，同时仍用 RL 的相对 advantage 抑制错误路径。

这也是为什么 VAPO 的贡献不是“回到 vanilla PPO”，而是为 long-CoT verifier reward 重做 value-based PPO。

GSPO arXiv:2507.18071v2 Qwen Team, Alibaba

Group Sequence Policy Optimization

GSPO 的核心批评是：GRPO 在 token 级别使用 importance ratio，但 reward 是 sequence 级别的， 两者粒度不匹配，长 response 会累积高方差噪声，MoE 的 routing volatility 还会放大 token ratio 波动。

$$s_i(\theta)=\left(\frac{\pi_\theta(y_i|x)}{\pi_{\theta_{\mathrm{old}}}(y_i|x)}\right)^{1/|y_i|} =\exp\left(\frac{1}{|y_i|}\sum_t \log\frac{\pi_\theta(y_{i,t}|x,y_{i,<t})}{\pi_{\theta_{\mathrm{old}}}(y_{i,t}|x,y_{i,<t})}\right).$$

机制原理

粒度错配

GRPO 的 reward 通常是整条答案的 outcome reward：这条 response 对或错。但优化时却对每个 token 单独计算 importance ratio 和 clipping。

这会把一个 sequence-level 判断拆成许多 token-level 约束。长答案上的 ratio 噪声会累积，某些 token 被 clip 后还会破坏整条轨迹的梯度一致性。

GSPO 的核心判断是：既然 reward 是给整条 response 的，importance ratio 和 clipping 也应该在 response 粒度上定义。

具体说，GRPO 里同一条 response 的第 1 个 token 可能 ratio=1.2，第 2 个 token ratio=0.8，第 3 个 token ratio=1.05；即使这条 response 只有一个整体 reward，不同 token 仍会被不同 ratio 加权、不同 clipping 处理。

GSPO 改成先把整条 response 的新旧 likelihood 比值合成一个 s_i(theta)，再用这个同一个 s_i 乘整条 response 的 advantage。于是这条 response 内所有 token 的梯度共享同一个 sequence-level 权重。

Sequence ratio

GSPO 用整条 response 的 likelihood ratio，并取长度归一化的几何平均，得到 s_i(theta)。

几何平均的作用是把不同长度 response 放到可比较尺度上，否则长答案的概率乘积天然更小、波动更大。

这样 clipping 排除的是“整条回答相对旧策略已经太 off-policy”的样本，而不是某个局部 token 的瞬时波动。

MoE 稳定性

MoE 模型里，每个 token 的专家路由会随参数、精度和执行引擎轻微变化。GRPO 的 token ratio 依赖具体 token 的激活专家，routing volatility 会让 ratio 不稳定。

所以 Qwen 团队曾需要 Routing Replay 这类工程补丁来固定旧策略的路由。GSPO 只要求 sequence-level likelihood 在语言建模意义上稳定，更能容忍局部路由抖动。

因此，它能减少对 Routing Replay 的依赖。

GSPO-token

论文还给出 GSPO-token 作为扩展：当任务确实有 token-level 或 turn-level advantage 时，可以把 sequence-level ratio 思想推广到更细粒度。

但在普通 verifier-based reasoning 里，若只有最终对错 reward，直接使用 sequence-level objective 更符合奖励信号来源。

关键判断：GSPO 不一定是所有 dense model 小规模训练的最高分方法，但它解决的是更底层的目标粒度错误。 如果你训练的是大 MoE 或有 training/inference engine 精度差异，GSPO 的价值会明显高于一个普通 recipe trick。
DCPO arXiv:2509.02333v2 Baichuan

Dynamic Clipping Policy Optimization

DCPO 直接针对两个利用率问题：固定 token clipping 对低概率 token 不友好，以及组内同 reward 时 advantage 归零。 它的重点指标不是只看 AIME，而是 TCR 与 RUR。

和 GRPO 到底差在哪

如果只看 OTM，并拿 canonical GRPO 公式对比，二者确实很接近：都是先在单条 response 内做 token mean，再把 response 作为训练单位聚合。OTM 更像是防止实现退化成 DAPO/TLM 或 batch-level token flatten 的 loss-reduction 选择，不是 DCPO 相对 GRPO 的最大差异。

DCPO 真正区别主要有两点：第一，SAS 改 advantage 标准化，让当前组全对/全错时不再直接变成零梯度；第二，DAC 改 token clipping，让旧策略低概率但正 advantage 的 token 有更大的更新空间。

组件创新程度一句话定位
DAC核心改动把固定 clipping bound 改成按旧 token 概率自适应，主要解决低概率好 token 学不到。
SAS核心改动把只看当前 group 的 reward 标准化改成结合历史/累计分布，主要解决全对/全错组零梯度。
OTM实现/聚合口径不是新 reward 或新 objective；更像是坚持 response 内 token mean，避免实现退化成跨 response/batch token flatten。

机制原理

DAC

固定 clipping bound 对所有 token 一视同仁，但不同旧概率 token 的“可学习空间”并不一样。

高概率 token 即使被上界限制，仍然容易继续变大；低概率 token 若偶然出现在高 reward 轨迹里，普通上界给它的绝对提升非常小。

Dynamic Adaptive Clipping 根据旧策略概率调整 bound：越低概率、但被正 advantage 支持的 token，允许更大的 ratio；同时设置上限避免极端 ratio 把训练打爆。

它和 DAPO Clip-Higher 的方向相同，但更细：不是全局放宽上界，而是按 token 概率自适应放宽。

SAS

GRPO 的零梯度来自“当前 prompt 的当前 rollout 组没有 reward 差异”。

SAS 不直接丢弃这类样本，而是把当前 step 的 reward 标准化与历史累计 reward 分布做平滑。直觉上，它用历史分布提供一个更稳定的参照系。

即使当前组全对或全错，也可以从长期 reward 位置判断这些 response 是否仍然应该产生更新。因此 RUR 不会随着训练后期全对/全错组增多而快速掉到很低。

OTM

每个 response 会产生很多 token，每个 token 都有一个 policy loss。训练代码最后必须把这些 token loss 汇总成一个标量，这一步叫 loss reduction。

GRPO 公式是先在单条 response 内做 token mean，再对同组 response 做 mean；这会让每条 response 的总权重相同，但单个 token 的权重随 response length 变小。

DAPO 的 token-level mean 则把同组所有 token 展开后按总 token 数平均；这会让每个 token 权重相同，但长 response 的总权重更大。

DCPO 论文里的 OTM 强调只在单条 response 内做 token mean，避免跨 response 或 batch 的 token-level 平均改变不同 response 之间已经由 advantage 表达出的相对关系。

一个具体例子：同一 prompt 下 response A 的 advantage=1、长度 500，response B 的 advantage=0.5、长度 1500。若先忽略 ratio/clipping 差异，把每个 token 的基础项看成相同，DAPO/TLM 会让 A 的长度权重是 500/(500+1500)=0.25，B 是 0.75，于是 A 的总贡献是 1×0.25=0.25，B 是 0.5×0.75=0.375；B 明明 advantage 更低，却因为更长而贡献更大。

OTM 则先算每条 response 自己的 token mean：A 得到 1，B 得到 0.5，然后再把 response loss 交给后续 batch/microbatch 聚合。这样 A:B 的贡献关系仍接近 1:0.5，而不会被长度变成 0.25:0.375。

所以 OTM 不是新的 reward，也不是重新算 advantage；它是在 loss reduction 层面选择“谁被平均、谁不被平均”。

TCR / RUR

DCPO 的重要性在于提出了更贴近训练浪费的诊断指标。

TCR 看多少 token 因 clipping 被排除在有效更新外；RUR 看生成出来的 response 中有多少带非零 advantage、真正参与 policy update。

AIME 分数只能看终点，TCR/RUR 能解释为什么同样 rollout 预算下，有的方法其实在浪费梯度。

AReaL arXiv:2505.24298v5 · NeurIPS 2025 Tsinghua / Ant Group / HKUST

AReaL: Asynchronous RL System for Language Reasoning

AReaL 不是 GRPO objective 的小修，而是系统级答案：同步 RL 中 generation 必须等 batch 内最长 response，导致 rollout GPU 空转； AReaL 完全解耦 generation 与 training，让 rollout worker 流式持续生成。

机制原理

同步等待浪费

长 CoT 的 response length 分布高度长尾。同步 rollout 要等一个 batch 里最长的样本生成完，短样本对应的 GPU worker 已经空闲，trainer 又必须等完整 batch 才能更新。

AReaL 的核心不是改 reward，而是把 generation 和 training 从批处理锁步改成流式 pipeline。

这样 rollout worker 可以持续产出样本，trainer 也可以持续消费样本。

Interruptible rollout

异步系统中，rollout 正在生成时 policy 可能已经更新。若不中断，样本会越来越 stale；若每次更新都等当前生成结束，又退回同步瓶颈。

AReaL 在新权重到达时允许中断生成，丢弃旧 KV cache，用新权重重算 prefix 后继续 decode。

论文给出的关键论证是：这种被多个 policy 片段生成的 trajectory，仍可视为某个 behavior policy 下的样本，因此可以用 importance sampling 纳入 PPO。

Staleness control

异步训练的代价是样本由旧 policy 生成，和当前 policy 有分布差。

staleness 太小，系统吞吐回到同步；staleness 太大，PPO clipping 中心错位，最新 policy 会被旧样本拉偏。

AReaL 用最大 staleness 约束训练 batch，实质是在“吞吐提升”和“off-policy 偏差”之间设一个可控边界。

Decoupled PPO

标准 PPO 默认 behavior policy 和 proximal policy 是同一个旧策略。异步场景里，如果直接用很旧的 behavior policy 当 clipping center，会把当前模型往旧策略附近拉。

Decoupled PPO 把“样本来自谁”和“更新时靠近谁”拆开：behavior policy 用于重要性采样校正，proximal policy 用于限制当前更新步长。

这样异步样本可用，但不会主导优化方向。

05落地建议

小到中等规模 math RLVR

先用 DAPO 作为可复现 baseline；如果出现错误答案越来越长，优先引入 Dr. GRPO 的去长度偏置实现； 如果全对/全错组变多、有效 batch 下降，再看 DCPO 的 SAS 或 DAPO dynamic sampling。

追求上限且能承担 critic 成本

VAPO 是值得读和试的方向，但不要直接把 vanilla PPO 搬过去。value pretraining、decoupled GAE 和 length-adaptive GAE 是必要组件，不是可选优化。

MoE 与超大模型 RL

GSPO 比 DAPO/DCPO 更像 objective 粒度修正。MoE routing 会让 token-level ratio 抖动，sequence-level ratio 能绕开 Routing Replay 的额外复杂度。

GPU 小时成为瓶颈

AReaL 是系统侧必读。长 CoT 训练的吞吐瓶颈通常不是 optimizer step，而是 rollout 等待最长样本。异步化后必须配套 staleness 控制和 decoupled PPO。

推荐阅读顺序

  1. DAPO：建立 RLVR recipe 与常见训练症状的直觉。
  2. Dr. GRPO：理解为什么“长度变长”可能是 objective bug，而不是能力涌现。
  3. DCPO：把 zero-gradient、RUR、TCR 这些监控指标补齐。
  4. GSPO：从重要性采样粒度理解 token-level ratio 的根本问题。
  5. VAPO：在 value-free 之后看 value-based 如何重新做 credit assignment。
  6. AReaL：最后看系统，把 rollout 利用率和异步 stale data 问题串起来。
