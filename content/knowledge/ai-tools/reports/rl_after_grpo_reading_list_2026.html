DeepSeek V4 官方技术报告，作为系统级 follow-up
一句话结论： GRPO 之后的主流方向不是单纯换一个 RL 算法，而是在解决四个工程和优化问题： 长度偏置、同奖导致的 advantage 消失、token-level ratio 高方差、rollout 成本过高。 DeepSeek V4 则说明前沿模型已经把 RL 算法、专家后训练、on-policy distillation、长上下文推理系统和 sandbox rollout 基础设施绑在一起做。

推荐阅读路线

如果只想快速建立框架，按下面四层读；P0 必读，P1 选读，P2 泛读或作为查漏补缺。

层级 要解决的问题 优先读 读完应获得的判断
P0 起点 GRPO 为什么能替代 PPO 的 value model，DeepSeek-R1 怎样用纯 RL 激发 reasoning。 DeepSeekMath, DeepSeek-R1 理解 group-relative advantage、rule/verifiable reward、cold start + RL + rejection sampling + distillation 的基本范式。
P0 算法修正 GRPO 训练不稳定、长度偏置、零梯度、rollout 利用率低。 DAPO, Dr. GRPO, VAPO, GSPO, DCPO, REAL 知道每个变体在改 clipping、advantage、value/critic、粒度，还是改样本选择。
P1 训练配方 怎样复现 R1-Zero-like training；数据难度、训练步数、任务混合是否比算法更重要。 SimpleRL-Zoo, Open-Reasoner-Zero, Online Difficulty Filtering, 1-shot RLVR, ProRL 判断 RL 到底是在放大 base model 已有能力，还是能突破 sampling boundary。
P1 扩展 RLVR 从数学/代码扩到多模态、工具调用、agent、多领域 reasoning。 Nemotron-CrossThink, SRPO, rStar2-Agent, SimpleTIR, Agent-R1 理解 reward 从最终答案走向过程、工具环境、trajectory 和 sandbox。
P2 系统化 前沿模型如何把 RL 与架构、长上下文、推理基础设施耦合。 DeepSeek V4, Survey of RL for LRMs, Agentic RL Survey 看到下一阶段不是“只调 PPO/GRPO”，而是长期 rollout、在线学习、agent 环境和基础设施协同。

DeepSeek V4 里值得看什么

DeepSeek V4 不是一篇 arXiv 论文，而是 2026-04-24 发布的官方模型和技术报告。 它对 RL 方向最有价值的点，是把 GRPO 后训练放进完整系统：领域专家、OPD 合并、百万上下文 rollout、agent sandbox。

1. 架构和优化

V4-Pro 是 1.6T 参数、49B activated 的 MoE，V4-Flash 是 284B 参数、13B activated； 两者支持 1M context。核心升级包括 Hybrid Attention，也就是 CSA + HCA， mHC 残差连接，以及 Muon optimizer。

CSA/HCA mHC Muon FP4 + FP8

2. 后训练：Specialist + OPD

报告写到：每个目标域先训练一个 expert，流程是 domain SFT 后再用 GRPO 做 RL， reward model 对齐各自成功标准；之后用 multi-teacher on-policy distillation 把十多个专家合并进单一模型。

GRPO domain experts reverse KL OPD

3. RL/OPD 基础设施

重点不是算法公式，而是可扩展 rollout：FP4 加速 rollout 和 teacher/reference forward； full-vocabulary OPD 通过 teacher hidden state 缓存避免显式 materialize 巨大 logits； rollout 服务用 token-granular WAL 做抢占安全恢复，避免中断导致的长度偏置。

fault-tolerant rollout full-vocab OPD WAL

4. Agentic AI sandbox

V4 报告引入 DSec sandbox 平台，用统一 Python SDK 支持 function call、container、 microVM、fullVM，并维护 trajectory log，服务于 post-training 和 evaluation 的工具执行环境。 这和 rStar2-Agent、Agent-R1、SimpleTIR 的方向相互印证。

sandbox tool trajectory deterministic replay
阅读建议： V4 报告不要当作“新 RL 算法论文”读。它更像是下一代 reasoning model 的系统蓝图： 长上下文降低 rollout/agent 成本，RL 培养专家能力，OPD 做能力合并，sandbox 支撑工具任务。

P0GRPO
DeepSeek-R1: Incentivizing Reasoning Capability in LLMs via Reinforcement Learning

把 GRPO 推到大规模 reasoning：R1-Zero 证明纯 RL 能涌现反思、验证和长 CoT；R1 加入 cold-start、rejection sampling、SFT 和 distillation 解决可读性与通用能力。

指出 GRPO 的 difficulty bias 和 response length bias，提出 Dr. GRPO 纠偏；也讨论 base model 预训练偏置和 prompt template 对 R1-Zero-like 训练的影响。

P0Dr. GRPObias
VAPO: Efficient and Reliable Reinforcement Learning for Advanced Reasoning Tasks
2025-04 · Value-based Augmented PPO

重新引入 value-based 路线，处理 value model bias、异构长度、稀疏 reward 等问题。适合和 DAPO/GRPO 对照，理解 critic-free 与 value-based 的取舍。

P0value-based
Group Sequence Policy Optimization
2025-07 · Qwen team · GSPO

把 GRPO 的 token-level importance ratio 改成 sequence-level ratio 和 clipping。重点是降低高方差、稳定 MoE RL 训练；据论文称贡献到 Qwen3 系列提升。

P0sequence-levelMoE
DCPO: Dynamic Clipping Policy Optimization

Rewards as Labels: Revisiting RLVR from a Classification Perspective
2026-02 · REAL

把 verifiable reward 从 scalar weight 改看成分类标签，缓解 positive gradient misassignment 和 negative domination。是 2026 年值得跟进的新范式。

P1classification view
TreeAdv: Tree-Structured Advantage Redistribution for Group-Based RL

把 group rollouts 显式组织成树，对共享路径和分叉路径做 advantage redistribution，目标是减少冗长 CoT 和提高 token 利用率。

用 smooth temperature-controlled gate 替代硬 clipping，在 sequence coherence 和 token-adaptive 之间折中。可作为 GSPO 后续读。

Dynamic Hybrid Policy Optimization for RLVR

读完后应抓住的趋势

1. 从 outcome reward 到更细粒度信号

GRPO/RLVR 最初依赖最终答案是否正确。后续方法开始处理过程一致性、reflection、tool feedback、tree segments、subsentence/token 等更细粒度 credit assignment。

2. 从 token ratio 到 sequence/subsequence/hybrid ratio

GRPO 高方差来自 token-level ratio；GSPO 改 sequence-level；SSPO/DHPO/SAPO 等继续寻找稳定性和信用分配之间的折中。

3. Data selection 可能和算法同等重要

DAPO dynamic sampling、Online Difficulty Filtering、Scaf-GRPO、1-shot RLVR 都说明：让训练样本产生非零且有区分度的 reward，比单纯换 optimizer 更关键。

4. 长期 RL 和 agent 环境会成为下一阶段

DeepSeek V4 的 DSec、rStar2-Agent、Agent-R1、SimpleTIR 都指向同一件事：模型不只是写答案，而是在长期环境里执行、观察、恢复、复盘。

5. “RL 是否创造新能力”仍未定论

Critical papers 认为 RL 多半重排 base distribution；ProRL 等认为足够长的稳定 RL 能扩展 reasoning boundary。这个争论会影响你如何评估训练效果。

6. Frontier model 的瓶颈是系统协同

V4 报告里 RL 只是后训练的一层；长上下文 attention、KV cache、FP4 rollout、teacher scheduling、sandbox、fault tolerance 都会影响最终上限。

建议阅读计划

第 1 天
建立基线： DeepSeekMath 只读 GRPO 部分；DeepSeek-R1 读 R1-Zero、GRPO、R1 pipeline。目标是能手写 GRPO objective，并说清和 PPO 的差异。
第 2-3 天
算法核心： DAPO、Dr. GRPO、VAPO、GSPO 精读；DCPO、REAL 泛读。目标是整理一张表：每个算法改的是 clipping、advantage、reward shaping、value，还是 data sampling。
第 4-5 天
训练 recipe： SimpleRL-Zoo、Open-Reasoner-Zero、Online Difficulty Filtering、1-shot RLVR、ProRL。目标是理解复现 R1-like training 时哪些细节比算法名更重要。
第 6 天
扩展到 agent： rStar2-Agent、SimpleTIR、Agent-R1、Agentic RL Survey。目标是理解 multi-turn tool trajectory 怎么定义 state/action/reward。
第 7 天
系统视角： DeepSeek V4 技术报告读 Architecture、Post-Training、RL/OPD Infrastructure、Sandbox Infrastructure 四节。目标是看到下一代 RL reasoning model 的全栈形态。
